# HashBench

**HashBench** — это легковесный и расширяемый C++ фреймворк (модульный движок) для бенчмаркинга алгоритмов хэширования. Проект построен на базе плагинной архитектуры, позволяющей изолированно добавлять новые хэш-функции, наборы данных (датасеты) и метрики оценивания без изменения логики самого ядра.

Фреймворк выполняет комплексный анализ распределения хэшей, оценивает скорость их работы, коллизии, энтропию и выводит консолидированный взвешенный результат (Final Score) на основе линейной и геометрической свёрток.

---

## Архитектура фреймворка

Проект разделен на изолированное ядро (`hashbench_framework`) и CLI-утилиту, которая собирает компоненты воедино. Расширение системы происходит через реализацию трех ключевых интерфейсов:

*   **`IHash`** — контракт для интеграции алгоритмов хэширования.
*   **`IMetric`** — контракт для создания метрик оценки (скорость, качество распределения и т.д.).
*   **`IDataset`** — контракт для загрузки тестовых данных.

Все созданные компоненты регистрируются в центральном фабричном реестре **`Registry`**, доступ к которому осуществляется через конфигурационный файл YAML.

## Требования и зависимости

Для сборки проекта вам понадобятся:
*   Компилятор с поддержкой **C++17** (например, GCC 9+, Clang 9+)
*   **CMake** версии 3.14 или выше
*   Библиотека **yaml-cpp** (для парсинга конфигурационных файлов)

### Установка зависимостей в Ubuntu:
```bash
sudo apt update
sudo apt install build-essential cmake libyaml-cpp-dev
```

Сборка осуществляется стандартным для CMake способом:

```Bash
mkdir build && cd build && cmake .. && make
```

После успешной компиляции в папке build появится исполняемый файл hashbench.

Конфигурация (Файл настроек)
Запуск бенчмарка полностью управляется через конфигурационный файл в формате YAML. Фреймворк гибко настраивает веса метрик и параметры тестирования.

Пример файла конфигурации config.yaml:

YAML
```
# Веса метрик для расчета итоговой оценки (сумма весов не обязана быть равной 1.0)
weights:
  collision: 0.4
  ns_per_op: 0.3
  entropy: 0.2
  pearson: 0.1

# Список хэш-функций, которые будут участвовать в тестировании
hashes:
  - CRC32
  - MurmurHash3
  - XXHash
  - CityHash
  - SHA256

# Коэффициент смешивания (лямбда) для расчета итогового балла. 
# final_score = lambda * sum_score + (1 - lambda) * geom_score
lambdas:
  - 0.5

# Факторы заполнения (Load Factors) хэш-таблицы для расчета размера бакетов
load_factors:
  - 0.5
  - 0.75

# Список тестируемых датасетов
datasets:
  - path: "data/words.csv"
    key: 0   # Индекс колонки в CSV, которая будет использоваться как ключ хэширования
  - path: "data/urls.csv"
    key: 1
```
 Использование утилиты
Запустите скомпилированный бинарный файл, передав ему путь к файлу конфигурации:

```Bash
./hashbench path/to/config.yaml
```
Пример вывода в консоль:
```
Results saved to: output/run_2026-05-15_11-30-00/run_2026-05-15_11-30-00.csv
```

 Результаты тестирования (Вывод данных)
При каждом запуске фреймворк автоматически создает уникальную поддиректорию в папке output/ с временной меткой: output/run_YYYY-MM-DD_HH-MM-SS/.

Внутри создается .csv файл, содержащий подробную аналитику:
```
Колонка	Описание
lambda	Использованный коэффициент сбалансированности оценок.
load_factor	Текущий коэффициент заполнения таблицы при замере.
dataset	Путь к файлу с данными.
hash_name	Название алгоритма хэширования.
raw_*	Сырые (реальные) значения метрик (например, время в нс, процент коллизий).
norm_*	Нормализованные метрики в диапазоне [0, 1] (где 1 — лучший результат, 0 — худший).
sum_score	Линейная взвешенная сумма всех метрик.
geom_score	Геометрическая взвешенная оценка (строго штрафует за провал даже в одной метрике).
final_score	Итоговый интегральный рейтинг хэш-функции.
```
 Расширение фреймворка (Добавление компонентов)
Фреймворк легко дополнять новыми алгоритмами или метриками без изменения кода ядра.

1. Добавление нового хэша
Создайте заголовочный файл в cli/hash/ (например, MyCustomHash.hpp):

```C++
#pragma once
#include "core/IHash.hpp"

class MyCustomHash : public IHash {
public:
    std::string name() const override { return "MyCustomHash"; }
    uint64_t hash(const std::string& key) const override {
        // Ваша логика хэширования
        return 42; 
    }
};
```
2. Регистрация компонента
Зарегистрируйте ваш класс в функции registerAll() внутри файла cli/main.cpp:

```C++
#include "hash/MyCustomHash.hpp"

void registerAll() {
    auto& r = Registry::instance();
    // ... существующие регистрации ...
    r.registerHash("MyCustomHash", []() { return std::make_shared<MyCustomHash>(); });
}
```
После этого имя "MyCustomHash" станет доступно для использования в конфигурационном yaml-файле!
